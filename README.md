# fpy — обновление справочников в булочной

Маленькая утилита: подменяет файлы справочников в локальной папке булочной свежими копиями из этой же папки скрипта.

## Что в папке

- `_8.py` — сам скрипт.
- `_8.txt` — список имён справочников, по одному на строку. Пустые строки и строки с `#` игнорируются.
- `_8.log` — журнал работы, создаётся автоматически.
- `R09.adt`, `R09.ADI`, … — свежие версии файлов справочников (их и будем переносить).

## Куда вписать путь к своей булочной

Открой `_8.py`, **строка 21**:

```python
DST_DIR   = r"C:\fabius\ohc\REFLIS"
```

Замени путь в кавычках на путь к папке справочников в твоей булочной. Префикс `r` перед строкой обязателен — он отключает интерпретацию `\` как спецсимвола.

Пример для другой булочной:

```python
DST_DIR   = r"D:\bakery\ohc\REFLIS"
```

## Как это работает

Для каждого имени из `_8.txt` (например `R09`):

1. Пытается удалить все файлы `R09.*` в `DST_DIR`.
2. Если хоть один из этих файлов **захвачен** другим процессом (касса работает, кто-то открыл таблицу) — справочник целиком **пропускается**, в `DST_DIR` ничего не пишется. Это безопасно: либо обновление прошло полностью, либо не было совсем.
3. Если удаление прошло чисто — копирует `R09.*` из папки скрипта в `DST_DIR`.

## Запуск

```
python .\_8.py
```

Что увидишь в консоли (то же самое падает в `_8.log`):

```
К переносу: 1 -> ['R09']
[1/1] R09
  del  C:\fabius\ohc\REFLIS\R09.adt
  del  C:\fabius\ohc\REFLIS\R09.ADI
  cp   C:\fabius\ohc\REFLIS\R09.adt
  cp   C:\fabius\ohc\REFLIS\R09.ADI
ИТОГО: ok=1, пропущено=0
```

Если файл захвачен, в логе будет `LOCK ...` и `-> пропуск`.

## Журнал `_8.log`

Каждая операция дописывается в `_8.log` с датой и временем. Когда файл переваливает за **300 КБ**, при следующем запуске старая половина строк отбрасывается, и работа продолжается.

## Добавить новые справочники

Просто допиши имена в `_8.txt`, по одному на строку:

```
R09
R10
R11
```

Файлы `R10.*`, `R11.*` должны лежать в папке скрипта рядом с `_8.py`.

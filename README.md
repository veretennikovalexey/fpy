# Перенос справочников Advantage Database Server

## Как работает `_7.py`

Скрипт переносит произвольный список справочников (`.adt`-таблиц) из этой папки в словарную базу Advantage Database Server.

**Что задаёт пользователь:**

1. **Список справочников** — текстовый файл `_7.txt`, который лежит рядом с `_7.py`. По одному имени таблицы на строку, без расширения. Пустые строки и строки, начинающиеся с `#`, игнорируются. Пример содержимого:

   ```
   R09
   R11
   # это комментарий, пропустится
   R23
   ```

2. **Путь к словарю-приёмнику** — задаётся внутри `_7.py` в переменной `DST_CONN_STR`, строка `DataDirectory=...`. По умолчанию:

   ```
   DataDirectory=C:\fabius\ohc\REFLIS\DICT.ADD
   ```

   Если словарь лежит в другом месте — поменять эту строку.

3. **Путь к источнику** — папка со скриптом (`BASE_DIR`). В неё кладутся `freeadt.exe` и сами `.adt`-файлы справочников, перечисленные в `_7.txt`. Отдельно настраивать ничего не нужно — берётся автоматически.

**Что делает скрипт для каждой строки из `_7.txt`:**

1. Запускает `freeadt.exe -y <имя>.adt` — отвязывает таблицу от словаря, чтобы её можно было прочитать без него.
2. Подключается к локальной свободной таблице (`ServerTypes=2`, `TableType=ADT`) и читает все строки через `SELECT * FROM <имя>`.
3. Подключается к словарю-приёмнику (`ServerTypes=2`, через ALS), в одной транзакции выполняет `DELETE FROM <имя>` и `INSERT` всех прочитанных строк через `executemany`. При ошибке — `rollback`.

Если одна таблица упала — выводится ошибка, и скрипт идёт к следующей. В конце печатается сводка: сколько перенесено успешно и какие справочники упали с какой ошибкой.

**Запуск:**

```
python _7.py
```

## ВАЖНО

**Python-зависимости:**

```
pip install pyodbc
```

Версия Python — 3.x, других зависимостей кроме `pyodbc` нет (всё остальное — стандартная библиотека).

**ODBC-драйвер Advantage — обязательно поставить отдельно.**

Сам `pyodbc` ничего не умеет без установленного ODBC-драйвера. Нужно поставить **Advantage ODBC Driver** соответствующей разрядности — для 32-разрядной версии ADS это файл **`adsodbc.exe`** (32-bit). Без него `_7.py` (и `_4.py`, `_5.py`, `_6.py`) упадут на первом же `pyodbc.connect(...)` с ошибкой вида:

```
[IM002] Data source name not found and no default driver specified
```

**Разрядность критична:** 32-битный Python работает только с 32-битным ODBC-драйвером, 64-битный — только с 64-битным. Если ставишь `adsodbc.exe` 32-bit — запускай 32-битный Python.

**Как проверить, что драйвер установлен:**

`Win + R` → `odbcad32.exe` → вкладка **Drivers** → в списке должен быть `Advantage StreamlineSQL ODBC`.

На 64-битной Windows два администратора ODBC:
- `C:\Windows\System32\odbcad32.exe` — показывает 64-битные драйверы
- `C:\Windows\SysWOW64\odbcad32.exe` — показывает 32-битные драйверы

**Сервер Advantage ставить НЕ нужно.**

Все подключения в `_7.py` идут с `ServerTypes=2` — это **ALS (Advantage Local Server)**, встроенный прямо в ODBC-драйвер. Никакой `adsserver.exe` / службы Windows «Advantage Database Server» устанавливать не требуется: достаточно только `adsodbc.exe`.

Это касается и подключения к **словарю** (`DICT.ADD`), и подключения к свободным `.adt`-файлам — ALS открывает и то, и другое напрямую как файлы.

**Подводные камни ALS:**

- **Конкурентный доступ.** Если с этими же файлами параллельно работает другая программа (например, штатная учётная система), её надо закрыть на время переноса — ALS не координирует блокировки так же надёжно, как полноценный ADS server.
- **Сетевые шары.** ALS умеет работать с файлами на сетевой папке, но это медленно и нестабильно — лучше переносить локально.
- **Производительность.** На больших справочниках ALS заметно медленнее AIS, но для разовых переносов это не критично.
